telegram_messages is a Node.js serverless function designed to run inside Appwrite Functions, sending automated messages to a Telegram group using the Telegram Bot API. It provides a simple, self-contained way to push notifications from any backend event — no extra infrastructure required beyond an Appwrite project and a Telegram bot.

## How It Works

The function is triggered inside Appwrite — either via an HTTP call, a scheduled CRON expression, or an Appwrite event (such as a new document being created). Once triggered, it follows three steps:

<Steps>
  <Step title="Function is triggered">
    Appwrite invokes the function based on the configured trigger: an incoming HTTP request, a time-based schedule, or a database/storage event.
  </Step>
  <Step title="Read credentials from environment">
    The function reads `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` from Appwrite's encrypted environment variable store — no secrets are hardcoded in the source.
  </Step>
  <Step title="POST to the Telegram Bot API">
    A `fetch` call is issued to `https://api.telegram.org/bot<token>/sendMessage` with the target chat ID and the message body. Telegram delivers the message to the group instantly.

    ```javascript sendMessage.js
    const token = process.env.TELEGRAM_BOT_TOKEN;
    const chatId = process.env.TELEGRAM_CHAT_ID;

    const response = await fetch(
      `https://api.telegram.org/bot${token}/sendMessage`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ chat_id: chatId, text: message }),
      }
    );
    ```
  </Step>
</Steps>

## Environment Variables

<ParamField query="TELEGRAM_BOT_TOKEN" type="string" required>
  The token issued by BotFather when you create your Telegram bot. Format: `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`.
</ParamField>

<ParamField query="TELEGRAM_CHAT_ID" type="string" required>
  The numeric ID of the Telegram group or channel where messages will be sent. You can retrieve this by adding your bot to the group and calling the `getUpdates` API endpoint.
</ParamField>

## Deploying to Appwrite Functions

<Steps>
  <Step title="Create a new function in Appwrite Console">
    In your Appwrite project, navigate to **Functions → Create Function**. Choose **Node.js** as the runtime and configure the trigger type (HTTP, schedule, or event).
  </Step>
  <Step title="Set environment variables">
    In the function's **Settings → Environment Variables** section, add `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID`. Appwrite encrypts these values at rest.
  </Step>
  <Step title="Deploy the function code">
    Push the code to the function either by connecting the GitHub repository directly in the Appwrite Console or by using the Appwrite CLI:

    ```bash
    appwrite deploy function
    ```

    Once deployed, you can test the function by triggering it manually from the Console or by calling its HTTP endpoint.
  </Step>
</Steps>

<Note>
  To find your group's `TELEGRAM_CHAT_ID`, add your bot to the group, send any message, and then call `https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates`. The `chat.id` field in the response is the value you need.
</Note>
